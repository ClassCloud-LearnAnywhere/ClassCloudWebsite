Features

Everything you need to manage
virtual machines at scale

From student dashboards to advanced admin analytics, ClassCloud has it all.

Browser-Based Console

Access VMs directly from your browser using Apache Guacamole. No VPN or client software required.

SSO Integration

Seamless single sign-on with CAS and SAML support. Integrate with your existing university authentication.

Auto-Shutdown

Automatically shutdown idle VMs after 3 hours with live countdown timers. Save resources and reduce costs.

Usage Analytics

Real-time dashboards with usage heatmaps, activity tracking, and comprehensive audit trails.

Email Notifications

Automated email alerts for VM provisioning, system status, and important events. AWS SES integration included.

Role-Based Access

Granular permissions for students, professors, and admins. Secure multi-tenant architecture.

Bulk Operations

Manage hundreds of VMs at once. Bulk start, stop, provision, and email all students in a course.

Export & Reports

Export usage data, audit logs, and activity reports. CSV format for easy analysis and compliance.

Health Monitoring

Real-time health checks for Proxmox, database, Guacamole, and SMTP. Get alerts before issues arise.

Custom Branding

Customize the portal with your university's logo and colors. White-label ready for institutions.

Docker Deployment

Deploy in minutes with Docker Compose. Includes PostgreSQL, Guacamole, and reverse proxy with SSL.

Everything you need to know about ClassCloud

ClassCloud is a virtual machine management platform designed specifically for educational institutions. It provides a web-based interface for students to access their VMs, while giving professors and administrators powerful tools to manage, monitor, and provision virtual machines at scale.

ClassCloud requires a Proxmox VE cluster for hosting virtual machines. The ClassCloud portal itself runs in Docker containers and can be deployed on any server with Docker and Docker Compose installed. You'll also need PostgreSQL for the database and optionally Apache Guacamole for browser-based console access.

ClassCloud is licensed software for educational institutions. Contact us for pricing and licensing options tailored to your institution's needs. You'll need to provide your own servers and infrastructure to run Proxmox and host the application.

With Docker Compose, you can have ClassCloud running in under 5 minutes. The repository includes comprehensive documentation, example configuration files, and setup scripts. Basic knowledge of Docker and Linux is helpful, but our step-by-step guides make it accessible even for those new to these technologies.

Yes! ClassCloud has built-in support for both SAML and CAS single sign-on protocols. This allows seamless integration with your university's existing authentication system, so students can use their campus credentials to access their VMs.

Absolutely! ClassCloud integrates Apache Guacamole to provide browser-based console access. Students can connect to their VMs using VNC or RDP directly from their web browser—no VPN or client software required.

VMs automatically shutdown 3 hours after being started to conserve resources. Students see a live countdown timer on their dashboard. The shutdown is graceful, giving VMs time to properly close applications. Usage hours are tracked for reporting purposes.

ClassCloud provides comprehensive analytics including usage heatmaps, activity tracking, connection logs, and audit trails. Administrators can export reports in CSV format for further analysis. Real-time dashboards show system health, active users, and resource utilization.

Yes! ClassCloud supports custom branding including logos, colors, and email templates. You can white-label the entire platform to match your institution's identity. All branding assets are easily configurable through environment variables and template files.
